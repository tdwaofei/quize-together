## 一、单选题样例

### 考点 1.1 变量定义

**题目：** 以下代码的输出是：

```cpp
int a = 10;
int* p = &a;
cout << *p;
```

A. `a 的地址`  
B. `10`  
C. `编译错误`  
D. `随机值`

<details>
<summary>点击查看答案与解析</summary>

**答案：B**

**解题过程：**
- `&a` 获取变量 `a` 的地址。
- `int* p = &a;` 定义指针 `p`，让它指向 `a`。
- `*p` 是解引用，访问 `p` 指向地址中的值，即 `a` 的值 10。
- 输出：`10`

**本题考点：** 指针的定义与解引用。

**复习要点：**
- `int* p = &a;` — 定义指针 `p`，指向变量 `a`。
- `*p` — 解引用，访问指针指向的值。
- `&a` — 取地址，获取变量的内存地址。
</details>

### 考点 数列求和与公式变形

**题目：** 已知 $1² + 2² + 3² + ... + n² = \frac{n(n+1)(2n+1)}{6}$，求 $2² + 4² + 6² + 8² + ... + 100²$ 的值。

A. `171700`  
B. `172700`  
C. `173700`  
D. `174700`

<details>
<summary>点击查看答案与解析</summary>

**答案：A** 

**解题过程：**

将所求式子中的每一项提取公因数：

$$2² + 4² + 6² + ... + 100²$$

$$= (2 \times 1)² + (2 \times 2)² + (2 \times 3)² + ... + (2 \times 50)²$$

$$= 2² \times 1² + 2² \times 2² + 2² \times 3² + ... + 2² \times 50²$$

$$= 4 \times (1² + 2² + 3² + ... + 50²)$$

这里 $n = 50$，代入平方和公式：

$$1² + 2² + 3² + ... + 50² = \frac{50 \times 51 \times (2 \times 50 + 1)}{6}$$

$$= \frac{50 \times 51 \times 101}{6}$$

$$= \frac{257550}{6}$$

$$= 42925$$

因此原式结果为：

$$4 \times 42925 = 171700$$

**本题考点：** 平方和公式的应用与代数变形。

**复习要点：**
- 平方和公式：$1² + 2² + ... + n² = \frac{n(n+1)(2n+1)}{6}$。
- 偶数通项：$2, 4, 6, ..., 100$ 可表示为 $2k$，其中 $k = 1, 2, ..., 50$。
- 提取公因数：$(2k)² = 4k²$，将偶数平方和转化为 $4$ 倍的正整数平方和。
- 项数确定：从 $2$ 到 $100$ 的偶数共有 $100 \div 2 = 50$ 项。
- 常用求和公式：
  - 等差数列：$1 + 2 + ... + n = \frac{n(n+1)}{2}$
  - 平方和：$1² + 2² + ... + n² = \frac{n(n+1)(2n+1)}{6}$
  - 立方和：$1³ + 2³ + ... + n³ = \left(\frac{n(n+1)}{2}\right)^2$
</details>

## 二、多选题样例

### 考点 2.3 数据类型

**题目：** 以下哪些是 C++ 的基本数据类型？

A. `int`  
B. `string`  
C. `double`  
D. `char`

<details>
<summary>点击查看答案与解析</summary>

**答案：ACD**

**解题过程：**
- `int` 是整数类型，属于基本数据类型。
- `string` 是 STL 中的类，不是基本数据类型。
- `double` 是双精度浮点数，属于基本数据类型。
- `char` 是字符类型，属于基本数据类型。

**本题考点：** C++ 基本数据类型的识别。

**复习要点：**
- 基本数据类型：`int`, `char`, `float`, `double`, `bool`, `void` 等。
- `string` 是类类型，需要包含头文件 `<string>`。
- 自定义类型如 `struct`、`class` 不属于基本数据类型。
</details>

## 三、判断题样例

### 考点 3.2 数组越界

**题目：** 以下代码执行后，数组访问不会越界。

```cpp
int arr[5] = {1, 2, 3, 4, 5};
cout << arr[5];
```

A. `正确`  
B. `错误`

<details>
<summary>点击查看答案与解析</summary>

**答案：错误**

**解题过程：**
- 数组 `arr` 的有效下标范围是 0 到 4。
- `arr[5]` 访问的是第 6 个元素，已经越界。
- 越界访问会导致未定义行为（UB），可能输出随机值或程序崩溃。

**本题考点：** 数组下标范围的理解。

**复习要点：**
- 数组下标从 0 开始，到 `n-1` 结束。
- 访问 `arr[n]` 会越界。
- 数组越界是常见的编程错误，务必注意边界检查。
</details>
